# Sakura 类人记忆系统 · 权威方案

> 本文是记忆系统的**权威设计方案**，取代此前所有旧版。配套：`MEMORY_DESIGN_ALIGNMENT.md`（讨论纪要）、`HUMANIZATION_PLAN.md`（情绪/关系）、`TASK_ROUTING_PLAN.md`（任务路由）、`PROJECT_VISION_OVERVIEW.md`（总纲）。
>
> **第一性原理**：参考人类记忆机制，但**不照搬**，一切从我们的场景出发——
> **一个天各一方的亲密陪伴对象，通过文字/语音/视频/屏幕分享生活，来了解你、关心你、和你累积一段关系。**
>
> **设计准则**：简约、直击本质、不过度设计；能本地/规则解决就不调云端；隐私优先；现有家底之上改进，不推翻。

---

## 一、总架构：三层 + 两个 LLM 角色

```
感知源：文字对话 / 语音 / 屏幕截图 / (未来)摄像头 / 环境信号
        │
 ① 感官记忆（本地视觉）  99%用完即弃，只留"当前活动"瞬时状态；1%够格片段→一句经历↓
        │
 ② 工作记忆 = 当前对话上下文   零改动·不存·不额外LLM
        │  对话自然结束 / 空闲回顾 → 由【记忆提炼】总结
        ▼
 ③ 长期记忆  Qdrant 向量库 + 核心档案JSON
     6类：核心档案 / 近况 / 共同经历 / 性格模式 / 约定纪念日 / 情感转折
     提炼 · 时效更新 · 选择性遗忘 · 复合检索

两个 LLM 角色：
· 对话/困难任务 → 云端大模型（用户可见的高质量对话、重要片段深度提炼）
· 后台高频任务 → 本地小模型（视觉描述、日常记忆提炼、分类；省钱·隐私·快）
```

三层职责一句话：
- **感官记忆**：她"看见"你在做什么（本地视觉，几乎不存）。
- **工作记忆**：她"正在聊"的上下文（零改动）。
- **长期记忆**：她"记住"的关于你和我们的一切（LLM 提炼后存 Qdrant）。

> 与旧版的区别（已废弃的过度设计）：**删除**"秒级短期记忆(STM)独立层""短期记忆 SQLite 库""工作记忆作为独立对象（焦点/容量/惯性）"。工作记忆就是对话上下文，不新建东西。

## 二、感官记忆（重点新功能）

### 2.1 目标效果
像异地情侣分享生活那样：她能感知你的生活动态（在看剧/加班/打游戏/熬夜），并**自然融进关心**——"今天那部剧好看吗？""又熬夜了，眼睛不累？"。**不是截图存档，是把"看见"变成"在意"。**
感知来源可扩展：屏幕 +（未来）摄像头 + 环境信号（时间、活动/空闲）。

### 2.2 三级漏斗（省 token 的核心）
```
定时采样(屏幕，默认 10~15s，可配)
  │ 第0级 隐私门（最先）：命中隐私规则 → 连截图都不做
  │ 第1级 变化检测（帧差/pHash，纯CPU，零成本）：无实质变化 → 丢弃（砍 80%+）
  │ 第2级 本地视觉编码（GPU本地，不花API token）：小VLM 出一句话场景描述
  │        （可选前置 CLIP/SigLIP 先判场景大类，更省）
  │ 第3级 注意力门控（规则）：场景是否有意义切换？时长够？用户在说话/该关心？
  ├─ 否：仅更新内存"当前活动"瞬时状态（不惊动任何 LLM）
  └─ 是：凝成一句经历 → 交长期记忆提炼流程
```

### 2.3 本地视觉模型选型与部署
| 角色 | 推荐 | 显存(Q4) | 说明 |
|---|---|---|---|
| 变化/场景大类（可选前置） | CLIP / SigLIP | <1G | 非生成，判场景切换，最省 |
| **主力：场景描述 VLM** | **Qwen3-VL 4B**（次选 2B） | 4B≈8G / 2B≈4G | 专长屏幕理解、OCR、中日文，可量化 |
| 备选 | Moondream2(1.8B)轻 / MiniCPM-V 4.x(8B)强 | 低 / ~8G | 按显存权衡 |

- **部署**：首选 **Ollama**（一条命令、OpenAI 兼容 API）；要保证 JSON 用 **llama.cpp + GBNF 约束**。
- **是否训练：第一版不需要**。看图说一句话靠 prompt+few-shot+约束解码即可稳定；仅当需要特定口吻或反复出错才后期 QLoRA（Unsloth/LLaMA-Factory），列为可选优化，不进第一版。

### 2.4 感官记忆的存储
- **不存原始截图/帧/摄像头画面**（隐私+空间+无意义）。
- 内存只保留一个滚动覆盖的**"当前活动快照"**：`{在做什么, 场景大类, 起止时间, 置信度}`，不落盘。
- 只有"够格的一段活动"→ 凝成一句自然语言经历，交长期记忆流程。
- 一句话：感官层 99% 用完即弃，只留瞬时"当前活动"，1% 转文字上行。

### 2.5 隐私设计（必须有，陪伴场景的雷区）
在现有 `app/perception/privacy.py::PrivacyGuard`（进程/标题黑名单）基础上扩展为**用户可控三重闸**：
1. **全局总开关**：一键开/关屏幕感知，关闭时完全不采样。
2. **窗口/应用选择（用户拍板）**：白名单模式（只感知勾选的应用，更保守，推荐默认）/ 黑名单模式（默认感知但排除敏感窗口，沿用现有默认）。
3. **实时隐私门**：每次采样前检查活动窗口，命中即跳过（连截图都不拍）。
4. **感知状态提示**：桌宠有明确"正在感知/未感知"可见状态，杜绝被监视感。
5. **落库脱敏**：视觉提炼的经历进长期库前过一遍现有敏感信息过滤。

### 2.6 摄像头（第一版后置）
比屏幕更敏感。第一版**先只做屏幕**；摄像头作为后续扩展，用轻量存在/活动检测而非持续 VLM，默认关闭、独立授权。

## 三、工作记忆（零改动）
- **就是当前对话上下文**，LLM 读原文即可，不额外调 LLM、不单独存储。
- 沿用现有 `context_trimming`（保留最近 N 条）。
- 长对话逼近上下文上限时的摘要，**归入长期记忆提炼流程**（顺手沉淀"刚才聊了啥"），工作记忆层不做任何新东西。

## 四、长期记忆（重点）

### 4.1 设计原点：现实中人怎么记住一个异地对象
（详见 `MEMORY_DESIGN_ALIGNMENT.md` 第三节）要点：情绪决定记忆深浅；记"要义+感觉"而非逐字；尤其记住对方主动/脆弱的表露；心里维护一份"对方近况模型"并主动追问；选择性遗忘正常，但忘记重要的事像背叛；**在意 = 记住 + 回应 + 追问**。

### 4.2 记什么：6 类（已确认）
| # | 类别 | 内容 | 可变性 | 遗忘 |
|---|---|---|---|---|
| 1 | **核心档案** | 名字、称呼、家人朋友、职业、稳定喜好 | 稳定 | **豁免** |
| 2 | **近况/当前状态** | 最近在忙什么、心情、压力、正发生的事 | **常变** | 自然过期 |
| 3 | **共同经历 & 内部梗** | 一起经历/聊过的难忘片段 | 追加 | 情感浓的长留，琐碎淡出 |
| 4 | **性格/作息模式** | 习惯、脾气、作息（部分来自感官感知） | 缓慢演化 | 低衰减 |
| 5 | **约定/期待/纪念日** | 承诺、计划、纪念日 | 到点消解 | **豁免**（到点要想起） |
| 6 | **情感转折点** | 吵架又和好、脆弱坦露、重要表白 | 追加 | **豁免**（记最牢） |

**映射现有 Qdrant 分层**：1→`semantic`+`core_profiles.json`；2→`semantic`（标记可变，见 4.5）；3/6→`episodic`；4→`procedural`；5→`task` 复用但语义改为"约定/纪念日"，与 `reminders` 打通到点提醒。**传统 todo 弱化。**

### 4.3 存储
- **向量库**：沿用 **Qdrant**（本地 `on_disk`，`data/memory/qdrant/`，384维，Cosine）。
- **嵌入模型**：沿用 `paraphrase-multilingual-MiniLM-L12-v2`（多语言、本地、CPU 也能跑）。
- **核心档案**：沿用 `core_profiles.json`（必注入、不检索、不遗忘）。
- **访问痕迹**：沿用 `access_tracker.json`。
- metadata 扩展（在现有基础上加）：`emotion`(编码情绪)、`volatile`(是否可变近况)、`valid_until`(时效，可空)、`event_time`(事件发生时间)。

### 4.4 怎么提炼（本地为主，重要/困难上云端）
LLM 分工（混合，已确认）：
| 任务 | 频率 | 用哪个 | 理由 |
|---|---|---|---|
| 视觉场景描述 | 高 | 本地 VLM | 省钱·隐私·可高频 |
| 日常记忆提炼/去重/分类 | 中（每段对话后） | 本地小 LLM（Qwen3 4B/8B） | 后台批处理，省钱·隐私 |
| 复杂/重要片段深度提炼、反思 | 低 | 云端大模型 | 质量高，量少 |
| 用户可见对话 | 实时 | 云端大模型 | 体验第一 |
| 路由/该不该记/重不重要 | 高 | 规则 + 本地 | 近乎零成本 |

- **本地提炼 LLM**：Qwen3 4B/8B（Q4_K_M GGUF，Ollama）。同样第一版免训练，靠 prompt+schema+约束解码；后期不足再 QLoRA。
- **提炼流程**：输入这段对话/经历 + 相关现有记忆（控制注入量省 token）→ 本地 LLM 按 6 类抽"要义+情感"（存要义不存逐字）→ 结构化 JSON（约束解码保证格式）→ 复用现有去重(0.92)/合并(0.78)/纠错 → 写 Qdrant（`infer=False` 直接存）打标签 → 判为"重要/高情绪/含承诺"则升级云端做高质量提炼/反思。
- **关键铁律**：能力自适应——无本地条件时，记忆提炼直接走云端，功能不打折（提炼频率低，云端完全可承担）。

### 4.5 时效更新（借 Zep 思想，轻量，不上知识图谱）
- 仅对可变近况（`volatile=true`）做"**新盖旧**"：写新近况时检索同主题旧近况 → 给旧的置 `valid_until=now`（标记为过去，不删），新的生效。
- 检索默认只返回未过期的近况；历史近况仅在明确回溯时可查。
- 稳定档案（核心/性格）不参与"新盖旧"，走正常合并纠错。

### 4.6 提炼时机（改机械触发为拟人）
- 弃"每 8 轮"，改**事件驱动**：
  - **一段对话自然结束**：用户静默 > N 分钟（可配，如 5~10 分钟）→ 触发本地提炼。
  - **空闲回顾**：长时间无互动 / 每日低谷期 → 触发反思（`MemoryReflector`，可合并为一个"离线整理"任务，像睡前回顾今天）。

### 4.7 怎么回忆（检索）
现有复合评分方向正确，保留并小幅增强：
```
composite = 0.55·relevance + 0.25·recency + 0.20·importance
          + access_boost + η·emotion_congruence     # 新增：情绪一致性
          + 近况优先加权(volatile 且未过期)          # 新增：近况触手可及
→ MMR(λ=0.7) 去冗（现有）→ _touch_accessed 强化被想起的记忆（现有）
```
额外线索触发：纪念日/约定到点主动浮现（类别 5，对接 `reminders`）。

### 4.8 遗忘（选择性、拟人、可控）
- **豁免遗忘**（绝不丢）：核心档案(1)、约定/纪念日(5)、情感转折(6)。
- **自然过期**：近况(2) 超保留窗口（默认 3~7 天可配）未被提及 → 滚出/降权。
- **缓慢淡化**：琐碎共同经历(3) 的 vividness 随"久未激活×低重要×低情绪"衰减，低于阈值转冷归档（可极慢召回），被重聊则回升。
- **默认不做"记错/失真"**：只衰减不出错（安全）；模糊化/记岔作为拟人化强度=full 的可选项，后期再议。

## 五、能力自适应与降级（兼容所有用户）

**核心原则：记忆系统分"必备核心"和"可选增强"。**
- **必备核心（所有用户）**：长期记忆的提炼、存储、检索、遗忘。**绝不依赖本地显卡**。
- **可选增强（看硬件）**：感官记忆（本地 VLM 看屏幕）。无合适硬件则**自动关闭**，不影响核心陪伴。

**两个本地模型必要性不同：**
| 本地模型 | 是否必须本地 | 无显卡时 |
|---|---|---|
| VLM（看屏幕） | 只能本地（隐私+高频） | **关闭感官功能** |
| 提炼 LLM（整理记忆） | 不必须（低频） | **走云端**，功能不打折 |

**硬件分档（启动自检 → 推荐 → 用户可手动覆盖）：**
| 档 | 硬件 | 感官 | 记忆提炼 | 嵌入 |
|---|---|---|---|---|
| A 高配 | ≥12G 显存 | 本地 VLM 全速 | **默认本地**（省钱），可云端 | 本地 |
| B 中配 | 6~12G | 本地 VLM 2B / 低频 | 云端（显存让给 VLM） | 本地 |
| C 低配/无显卡 | <6G / 核显 | **直接关闭**（已定，不做云端视觉） | 云端 | 本地CPU 或 云端 |
| D 纯云兜底 | 任意 | 关闭 | 云端 | 云端 |

- 嵌入模型仅 384 维、几百 MB，**CPU 也能跑**，长期记忆对无显卡用户天然友好。
- **优雅降级**：本地模型加载失败/Ollama 没装/显存不足 → 自动降档 + 友好提示，绝不崩、不卡主流程。

## 七、分阶段落地路线
| 阶段 | 内容 | 效果 |
|---|---|---|
| **M0** | 本地 LLM/VLM 接入（Ollama 端点 + 本地/云端路由客户端） | 打通"本地模型"这条腿 |
| **M1** | 长期记忆提炼改造：6 类、本地提炼、事件驱动、时效更新 | 记得更准、更像人、更省 |
| **M2** | 检索增强：情绪一致性、近况优先、纪念日触发；遗忘/冷归档 | 会"想起"、会"淡忘"、会"到点记得" |
| **M3** | 感官记忆：三级漏斗 + 本地 VLM + 当前活动状态 + 隐私开关 | 她能"看见"你的生活并自然关心 |
| **M4** | 情绪调制（接 `HUMANIZATION_PLAN` 的 PersonaState）：情绪进编码/检索 | 情绪真正参与记什么/想起什么 |
| **M4+（可选）** | 视觉/提炼模型 LoRA 微调；摄像头感知 | 质量与感知维度升级 |

## 八、仍需进一步明确（下一步探讨）
1. **本地资源占用**：VLM(4B)+提炼LLM(4-8B)+嵌入 同时跑，显存/内存是否够（用户 4060Ti 16G 可行）；是否"同一模型分时复用/按需加载"，或"一个 VLM 兼任文本提炼"（待实测决定，见 `TASK_ROUTING_PLAN` 讨论）。
2. **本地服务形态**：Ollama 常驻后台 vs 随桌宠启停；离线/未装如何优雅降级。
3. **"够格进长期"的量化阈值**：活动时长、场景白名单、情绪强度门限（先给保守默认）。
4. **提炼 prompt 与 schema 细化**：6 类抽取模板、few-shot、单次产出条数与 token 预算档位。
5. **"对话自然结束"判定**：静默阈值取多少分钟？是否结合话题变化。
6. **近况保留窗口**：默认 3 天 / 7 天？不同子类是否不同（心情几天，"在忙的项目"可能几周）。
7. **与情绪系统推进顺序**：记忆(M1-M3)先行、情绪(M4)随后，还是同步预留接口。

> 待以上明确后，进入接口与数据结构的详细设计，再写代码。
