# Sakura 记忆系统 · 设计对齐纪要

> 本文是"讨论与对齐"的活文档，记录我们对记忆系统的共识、调研结论、以及尚待明确的问题。
> 定位：先把方向和思路对齐清楚，作为后续精简修订 `MEMORY_ARCHITECTURE_PLAN.md` 的依据。**本阶段不写代码。**
>
> 第一性原理：**参考人类记忆机制，但不照搬**；一切从我们的真实场景出发——
> **一个天各一方的亲密陪伴对象，通过文字/语音/视频/屏幕分享生活，来了解你、关心你、和你一起累积一段关系。**

---

## 一、总体共识：记忆分三层（已对齐）

| 层 | 定义 | 是否存储 | LLM 参与 | 状态 |
|---|---|---|---|---|
| **感官记忆** | 通过截图/视频帧/摄像头等，感知主人在做什么、生活作息与动态 | 几乎不存原始数据，只留"当前活动"瞬时状态；有意义的片段转成文字上交长期 | **本地视觉小模型**（不动用主 LLM） | 重点新功能，见第二节 |
| **短期/工作记忆** | 当前对话上下文 | 不存（就是上下文窗口） | 不参与 | **零改动**（已定） |
| **长期记忆** | 用独立 LLM 对经历进行总结、提炼后存储 | Qdrant 向量库 + 核心档案 | **专门的提炼 LLM** | 重点，见第三节 |

补充共识：
- 长对话超出上下文时的摘要，**归入长期记忆的提炼流程**，工作记忆层不单独做。
- 现有家底（Mem0+Qdrant、复合评分检索、Curator/Reflector）站位不低，**在其上改进，不推翻重来**。
- 全程原则：**能本地/规则解决就不调云端主 LLM；简约、不过度设计。**

## 二、感官记忆：让桌宠"看见"主人的生活（重点功能）

### 2.1 我们要的效果

不是"截图存档"，而是像异地情侣那样：**她能感知你的生活片段（在看剧、在加班、在打游戏、深夜还没睡……），并自然地把这些融进关心和交流里**——"今天那部剧好看吗？""又熬夜了，眼睛不累吗？"。

感知来源可扩展：**屏幕（截图/视频帧）+ 摄像头 + 环境信号（时间、活动/空闲）**。

### 2.2 核心矛盾与解法

矛盾：持续感知 vs token/算力预算。解法：**三级漏斗——用便宜的本地手段层层过滤，只有极少数"值得记的瞬间"才升级处理。** 用户机器有 NVIDIA 显卡，可本地部署，这是关键前提。

```
定时采样(屏幕/摄像头)
   │
第0级  变化检测(帧差/pHash，纯CPU，零成本)
   │   画面没实质变化 → 丢弃(砍掉 80%+ 帧)
   ▼
第1级  本地视觉编码小模型(GPU，本地，不花API token)
   │   ├─ 超轻档：CLIP/SigLIP/Florence-2 → 打场景标签(视频/代码/网页/游戏)
   │   └─ 推荐档：小型VLM → 一句话描述("在看恋爱番，看得很投入")
   ▼
第2级  注意力门控(规则)：场景是否有意义地切换？用户是否在说话/需要关心？
   │   多数：只更新"当前活动"瞬时状态，不惊动主 LLM
   ▼
   少数"够格"的片段 → 转成一句自然语言经历 → 交长期记忆流程
```

### 2.3 本地视觉模型选型（2026 调研）

| 档次 | 候选 | 特点 | 显存 | 适用 |
|---|---|---|---|---|
| 超轻（第一道筛） | **CLIP / SigLIP** | 非生成，编码成向量/标签，做场景分类与变化检测 | 极低 | 第 0.5 级：判大类、去重 |
| 超轻（任务型） | **Florence-2**（0.23B/0.77B, 微软） | caption/OCR/检测，便宜的"画面里有什么" | 很低 | 廉价一句话摘要 |
| 极小 VLM | **Moondream2**(~1.8B)、**SmolVLM**(0.25~2.2B) | 极快、端侧、基础描述 | 低 | 低配/高频场景 |
| **推荐 VLM** | **Qwen3-VL 2B/4B** | **专长屏幕/UI理解与"视觉agent"**，OCR强，中日文好，可量化(GGUF/AWQ) | 2B~4G / 4B~8G | **屏幕感知首选** |
| 强 VLM | **MiniCPM-V 4.x**(~8B)、InternVL2.5 | 接近 GPT-4V 级、视频理解好 | ~8G+ | 追求质量、显存充足 |

**推荐组合**：第 0 级帧差 → （可选 CLIP/SigLIP 判场景切换）→ **Qwen3-VL 2B/4B 本地出一句话场景描述** → 规则门控。
**部署**：Ollama / llama.cpp(GGUF) / LMDeploy，量化后 2B~4B 在消费级 N 卡很轻松。
> 摄像头场景（识别作息/在不在）比屏幕更敏感，可先用轻量存在/活动检测，深度理解后置。

### 2.4 感官记忆的存储方式

- **不存原始截图/帧/摄像头画面**（隐私 + 空间 + 无意义）。
- 内存里只保留一个**"当前活动快照"**：`{在做什么, 场景大类, 起始/更新时间, 置信度}`，滚动覆盖。
- **只有"够格的一段活动"**（如"看了两个多小时恋爱番""连续加班到深夜"）才被提炼成**一句自然语言经历**，交给长期记忆流程（先进"近况"，反复出现才巩固成"他喜欢看恋爱番"这类稳定认知）。
- 一句话：**感官层 99% 用完即弃，只留瞬时"当前活动"，1% 转成文字向上走。**

## 三、长期记忆：像"记住一个异地恋人"那样去记（重点）

### 3.1 先想清楚：现实中，人是怎么记住 & 回忆一个异地对象的？

这是设计长期记忆的"第一性原理"。综合心理学与生活经验，人际/亲密关系记忆有这些机制（我们要模拟这些，而不是模拟数据库）：

**A. 记住（编码）的规律**
1. **情绪显著性决定深浅**：初次见面、冲突、里程碑、脆弱时刻的坦露——记得深；日常寒暄——很快淡忘。（情绪 = 记忆的放大器）
2. **自我表露加深亲密**（社会渗透理论）：关系是靠**双方逐层袒露越来越私人的信息**推进的。我们尤其记住对方**主动告诉我们的、脆弱的**东西。
3. **记的是"要义"不是"逐字"**（模糊痕迹理论）：逐字很快忘，**要义与情感基调长期留存**。所以不该存对话原文，该存"发生了什么 + 什么感觉"。
4. **共同经历 & 内部梗**：一起做过的事、只有你俩懂的梗，是关系身份的核心。

**B. 回忆（提取）的规律**
5. **线索触发**：情境、日期（纪念日）、地点、物件、情绪都会勾起回忆（编码特异性 + 情绪一致性）。
6. **反复回味会强化**："记得那次……"——重述让记忆更牢，也构建"我们的故事"。
7. **持续更新对方的"近况模型"**：异地关系里，人会在心里维护一份"对方最近怎样"（作息、在忙的项目、压力、社交圈），**每次聊天更新它，并主动追问**（"上次那个考试结果怎么样？"）——这恰恰是"在意"的证据。
8. **交互记忆（transactive memory）**：长期伴侣会形成"共享记忆系统"，彼此成为对方的外部记忆。

**C. 遗忘的规律**
9. **选择性遗忘是正常的**：细节褪色、要义与情感留存；忘掉琐事是人之常情，但**忘掉重要的事（名字、承诺、重大情绪事件）会像背叛**。

**D. 关系的本质**
10. **"在意" = 记住 + 回应 + 追问**：亲密感的核心是"被理解、被回应"，而这主要来自**记得并引用对方分享过的东西**。

### 3.2 由此推导：Sakura 长期记忆应该记什么

综合上面，人对一个亲密对象长期保留的，就这 6 类（我们据此定分类）：

1. **核心身份档案**：名字、称呼、家人朋友、职业、稳定喜好——**永不遗忘**。
2. **对方的近况/当前状态**：最近在忙什么、心情、压力、正在发生的事——**会更新、会过期**。
3. **共同经历 & 内部梗**：一起经历过的、聊过的难忘片段——情感浓的长期留。
4. **对方的模式/性格**：作息、习惯、脾气（部分来自感官感知的生活作息）。
5. **承诺与期待/未来计划**：你许诺过的、约定的、纪念日——**到点要能想起**。
6. **情感转折点**：吵架又和好、脆弱坦露、重要表白——记得最牢。

> 对照现有四层（semantic/episodic/procedural/task）：核心档案+性格→semantic/procedural；近况→新增"近况"处理；经历+情感转折→episodic；承诺/纪念日→需要**带时间点、可提醒**的处理（现有 task/reminders 可复用但要与记忆打通）。恋人场景里传统的"todo task"意义不大，**弱化为"约定/期待"**。

### 3.3 由此推导：Sakura 长期记忆该怎么"回忆"

现有复合评分（相关性+新近度+重要性+回忆次数+MMR）已经很接近人脑，方向对。要补的：
- **情绪一致性**、**日期/纪念日线索触发**（到了纪念日主动想起）。
- **近况优先**：聊天时"你最近怎样"应触手可及（对应机制 7 的"近况模型"）。
- **回味强化**：被重新聊起的记忆自动增强（`_touch_accessed` 已有钩子）。

### 3.4 由此推导：怎么"提炼"与"更新"

- **用独立 LLM 提炼**（你的要求）：从一段对话/一段经历里，按 3.2 的分类抽出"要义+情感"，而非存原文（对应机制 3）。
- **提炼时机**：倾向"一段对话自然结束 / 空闲时"回顾，而非机械每 8 轮（更自然、更省）。
- **时效更新**（借 Zep 思想的轻量版，不上知识图谱）：近况/可变事实要能**新的覆盖旧的、旧的标记为过去**（你搬家/换工作），避免矛盾堆积。
- **选择性遗忘**：琐碎近况自然过期；核心档案/承诺/重大情感**豁免遗忘**（对应机制 9，避免"像背叛"）。

## 四、落地前尚待明确的问题（需要我们逐条探讨）

### 关于感官记忆
1. **采样与触发**：屏幕截图频率？是常态低频采样，还是仅在特定条件（你在活动、或她要主动关心时）才采样？摄像头是否纳入第一版，还是先只做屏幕？
2. **隐私边界（陪伴场景的雷区）**：
   - 全局总开关 + "感知中"的明确提示（避免被监视感）？
   - 窗口/应用黑名单（密码、银行、私密内容永不看）？现有 `app/perception/privacy.py` 有雏形，如何扩展？
   - 感官提炼出的"生活片段"落入长期记忆前，是否要脱敏？
3. **本地模型选型确认**：先上 Qwen3-VL 2B/4B 走通，还是先用更轻的 Florence-2/CLIP 做场景标签验证漏斗？部署方式（Ollama？）你有偏好吗？
4. **视觉→记忆的"够格"标准**：什么样的活动值得被记（时长阈值？场景类型白名单？用户情绪/发声？）。

### 关于长期记忆
5. **提炼用哪个 LLM**：本地小 LLM（省、可高频、隐私好）还是云端主模型顺手做（质量高但花钱）？两者可否混合（日常本地、重要片段云端）？
6. **分类落地**：认可 3.2 的 6 类吗？现有四层 metadata 如何映射（是否新增"近况""约定/纪念日"的处理）？
7. **提炼触发**：从"每 8 轮"改成"对话自然结束/空闲回顾"，具体用什么信号判断"一段对话结束"？
8. **时效更新的粒度**：哪些字段算"可变近况"（心情、在忙的事、住址…），哪些算"稳定档案"（名字、家人…）？覆盖时旧值保留多久/如何标记过去？
9. **遗忘策略**：琐碎近况保留多久淡出？哪些类别绝对豁免遗忘（建议：核心档案、承诺、重大情感）——你认可这个豁免清单吗？
10. **成本红线**：提炼时给 LLM 喂多少现有记忆做对照去重、单次产出多少条，需要定一个 token 预算档位。

### 关于整体
11. **与情绪状态（PersonaState）的关系**：情绪是记忆的"放大器"和"提取线索"。记忆与情绪是**先做记忆再接情绪**，还是同步设计接口预留？
12. **多模态归一**：屏幕感知、对话、（未来）摄像头、语音，提炼进长期时是否统一成"自然语言经历"这一种表示（建议：是，简化一切）？
